
# Align Report Export with CLI Format

## Overview
Complete rewrite of `generateMarkdownReport` in `src/lib/exportMarkdown.ts` to match the CLI's canonical report structure. Also update `getVulnerabilityCounts` in `src/pages/Report.tsx` to include `gas`.

## Changes

### 1. `src/lib/exportMarkdown.ts` — Full rewrite of `generateMarkdownReport`

**Interface update:** Add `gas: number` to `VulnerabilityCounts`.

**Helper functions to add:**
- `generateReportId(dateStr)` — produces `SOL-YYYYMMDD-HHmmss` from `audit.created_at`
- `formatDate(dateStr)` — produces `"February 22, 2026 at 10:01 UTC"`
- `computeOverallRisk(counts)` — returns risk string based on severity counts

**New section order (replacing entire function body):**

1. **Title + metadata** — `# Solarizer Security Audit Report` with Project, Report ID, Generated date, Lines Analyzed, Engine Version
2. **Table of Contents** — dynamic numbered links, only including severity sections that have findings
3. **Executive Summary** — 6-row severity table (including Gas), optional Grade/Score rows, overall risk line
4. **Audit Scope** — contract count + per-file list from `scope_metadata` if available
5. **Methodology** — 4-step pipeline (Static, Semantic, Cross-Contract, Validation) + Severity Classification table
6. **Risk Summary** — overall risk + total findings/contracts summary
7. **Findings by severity** — `## [emoji] [Severity] Findings` headings, `### PREFIX-N: Title` format, plain code blocks with line numbers, no HTML, index resets per severity
8. **Verification Coverage** — kept as-is but heading changed to `## Verification Coverage` (no emoji)
9. **Disclaimer** — Eryonix Techlabs legal text with ToS link
10. **Footer** — plain `*Report generated by Solarizer Security Engine -- solarizer.io*`

**Key removals:** All `<div>`, `<details>`, `<summary>` HTML; Contact section; copyright line; Manual Review methodology step; severity description quotes; emoji prefixes on non-finding section headings.

**`downloadMarkdown` helper** stays untouched.

### 2. `src/pages/Report.tsx` — Add `gas` to vulnerability counts

Update `getVulnerabilityCounts` (line 73-83) to include:
```ts
gas: findings.filter(f => f.severity === "gas").length,
```

This ensures the counts object matches the updated `VulnerabilityCounts` interface.

### Files affected
- `src/lib/exportMarkdown.ts` (rewrite `generateMarkdownReport`, update interface)
- `src/pages/Report.tsx` (add `gas` to counts)
